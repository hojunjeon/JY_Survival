## 아키텍처 결정

| 무기 | 접근 방식 |
|------|---------|
| Python | Projectile에 `chainHops`, `chainRadius`, `hitEnemyIds` 추가. 충돌 시 체이닝 로직 main.js에 추가 |
| C/C++ | Projectile에 `isRailgun` 플래그. 발사 시 triggerScreenShake + 조준선 렌더. cooldown 1.0s |
| Java | JavaWeapon에 `blackholeTimer` 추가. 5초마다 정지 Projectile(`isBlackhole`) 소환 → main.js에서 흡입 + 폭발 처리 |
| Git | GitWeapon에 `branchPoint`, `mergeTimer`, `mergeReady` 상태 추가. main.js render에서 선 그리기 + 라인 데미지 |
| SQL | SQLWeapon에 `dropPhase`, `targetX/Y`, `blockY`, `dropTimer` 상태 추가. main.js render에서 표적 + 블록 그리기 |
| JavaScript | JavaScriptWeapon에 `tornado` 상태 추가. main.js update/render에서 토네이도 범위 처리 |

## 태스크 목록

### T1: Python — Chain Lightning
**파일**: `weapons/Python.js`, `entities/Projectile.js`, `main.js`
- [ ] Projectile.js: `chainHops=0`, `chainRadius=0`, `hitEnemyIds=null` 필드 추가
- [ ] Python.js: 3방향 → 단일 초록 투사체 (`chainHops: 2`, `chainRadius: 120`, `hitEnemyIds: new Set()`)
- [ ] main.js 충돌 루프: Python 투사체가 적 히트 시 체이닝 (chainHops>0 → 가장 가까운 미방문 적 탐색 → 새 투사체 push)
- [ ] 시각: 투사체 색상 `#44ff44` (초록), 크기 6×6

### T2: C/C++ — Railgun
**파일**: `weapons/C.js`, `entities/Projectile.js`, `main.js`
- [ ] C.js: cooldown 1.0s, damage 35로 증가. fire 시 `isRailgun: true` 플래그
- [ ] Projectile.js: `isRailgun` 필드 추가. render에서 isRailgun이면 긴 직사각형(width:3, height:24) 회색으로 그림
- [ ] main.js tryFireWeapon: C무기 발사 시 `triggerScreenShake(5, 0.15)` 호출
- [ ] main.js render: C무기 보유 시 플레이어 → lastDir 방향으로 점선 조준선 그리기 (길이 400)

### T3: Java — GC Blackhole
**파일**: `weapons/Java.js`, `entities/Projectile.js`, `main.js`
- [ ] JavaWeapon: `blackholeTimer=5`, 5초마다 `_pendingBlackhole=true` 신호
- [ ] Projectile.js: `isBlackhole`, `blackholeRadius=100`, `blackholeLifetime=2.0`, `_bhTimer` 필드 추가
- [ ] main.js: JavaWeapon의 blackhole 신호 감지 → 가장 가까운 적 위치에 정지 Projectile(`vx=vy=0`, `isBlackhole`) 생성
- [ ] main.js update: blackhole proj 주변 적을 중심으로 x,y 조금씩 이동 (흡입). `_bhTimer` 소진 시 isAreaEffect 폭발로 전환 후 deactivate
- [ ] main.js render: 블랙홀 위치에 검은 원 + 보라색 테두리 그리기

### T4: Git — Branch & Merge
**파일**: `weapons/Git.js`, `main.js`
- [ ] GitWeapon: `branchPoint=null`, `mergeTimer=0`, `mergeReady=false` 상태. fire() 시 branchPoint 저장, mergeTimer=3 시작
- [ ] GitWeapon update: mergeTimer 카운트다운 → 0 되면 mergeReady=true
- [ ] main.js tryFireWeapon: git.mergeReady 감지 → branchPoint ↔ player 연결선 위 적들 데미지(50) + floatingText "Merge Conflict Resolved!" + git.branchPoint=null 초기화
- [ ] main.js render: branchPoint 존재 시 주황 원 표시 + player까지 점선

### T5: SQL — DROP TABLE
**파일**: `weapons/SQL.js`, `main.js`
- [ ] SQLWeapon: `dropPhase='idle'`, `targetX`, `targetY`, `blockY`, `phaseTimer` 상태. fire() → 가장 가까운 적 위치를 target으로 설정, dropPhase='targeting', phaseTimer=0.8
- [ ] SQLWeapon update: targeting → 0.8s 후 dropPhase='dropping', blockY=-50. dropping → blockY += 600*dt. blockY >= targetY 시 dropPhase='landed', phaseTimer=0.3
- [ ] main.js update: SQL dropPhase==='landed' 감지 → 범위 120 내 적 전체 데미지(60) + triggerScreenShake(8, 0.3) + 화면 플래시 + dropPhase='idle' 초기화
- [ ] main.js render: targeting 시 표적 십자선(빨간). dropping 시 `DROP TABLE` 블록 그리기(파란 사각형 + 텍스트)

### T6: JavaScript — Tornado
**파일**: `weapons/JavaScript.js`, `main.js`
- [ ] JavaScriptWeapon: `tornado=null`. fire() 시 tornado 생성 `{x:playerX, y:playerY, radius:30, maxRadius:180, timer:2.5, dmgTimer:0}`
- [ ] JavaScriptWeapon update: tornado.radius 확장 (50/s), tornado.timer 감소. dmgTimer 0.3s마다 리셋 → `_pendingDmg=true`
- [ ] main.js update: JS tornado._pendingDmg 감지 → 반경 내 적 데미지(8). tornado.timer<=0 → tornado=null
- [ ] main.js render: tornado 존재 시 반투명 노란 원 + 회전 파티클 이펙트 (arc 여러 개)
