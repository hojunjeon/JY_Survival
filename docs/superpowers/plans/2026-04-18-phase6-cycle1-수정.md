## T1: 이벤트 화면 글자 배치 중앙 정렬

**파일**: `ui/EventModal.js` — `_renderTriggered()`

**문제**: `y = by + 36` 에서 시작해 아이템을 top-down으로 쌓아 상단에 몰림.

**수정 방법**:
- 레이아웃 항목들의 총 높이를 사전 계산
- `startY = by + (bh - totalContentHeight) / 2` 로 수직 중앙 배치
- 각 항목(divider 간격 포함) 높이 합산: 제목(36) + 구분선(20) + NPC블록(80) + 구분선(20) + 대사(22×n) + 상황(22) + 액션(22)

## T2: 몬스터 피격 플래시 범위 수정

**파일**: `entities/Enemy.js` — `render()` 의 hitFlash 블록

**문제**: 플래시 `fillRect`가 적의 실제 시각 크기보다 크게 보임.

**수정 방법**:
- `ctx.save()` → `ctx.beginPath()` → `ctx.rect(this.x - this.width/2, this.y - this.height/2, this.width, this.height)` → `ctx.clip()` → 플래시 fillRect → `ctx.restore()`
- clip 적용으로 스프라이트/렉트 영역 밖으로 오버레이가 번지지 않도록

## T3: 이벤트 화면 NPC 이미지 상반신만 표시

**파일**: `sprites/PixelRenderer.js`, `ui/EventModal.js`

**문제**: NPC 스프라이트 32×32 전신(scale 3 = 96×96)이 표시됨.

**수정 방법**:
- `PixelRenderer`에 `drawSpriteClipped(ctx, sprite, x, y, scale, rowCount)` 메서드 추가
  - `rowCount`번째 행까지만 렌더 (16행 → 상반신)
- `EventModal._renderTriggered()`에서 `drawSprite` → `drawSpriteClipped(ctx, sprite, bx+24, y, 3, 16)` 교체

## T4: E3 → E2 개명 + 무한 스폰 + 30초 생존 미션

**파일**: `systems/EventSystem.js`, `ui/EventModal.js`, `ui/HUD.js`, `main.js`, `systems/WaveSystem.js`

**수정 내용**:

### EventSystem.js
- `e3State` → `e2State`, `e3Kills` → `e2Kills`, `e3Elapsed` → `e2Elapsed`
- E3 트리거/클리어 내부 로직 → E2 로 rename
- 클리어 조건: `e3Elapsed >= 30` (기존 60초 → 30초) + 처치 미션 삭제
  - `e2State === 'active'` && `e2Elapsed >= 30` → cleared
- `notifyKill('env_error')` 처리 블록 → e2Kills 집계만 (클리어 조건에서 처치수 제거)

### WaveSystem.js
- `setEventMode('env_error', 3)` — 3번째 파라미터가 maxSpawn 이면 `Infinity`로 변경
- env_error 스폰 제한 제거 (무한 생성)

### EventModal.js
- `EVENT_CONFIG.E3` → `EVENT_CONFIG.E2`
- `EVENT_MESSAGES.E3` → `EVENT_MESSAGES.E2`
- `actionBase: 'EnvError를 처치하고 60초를 생존하세요'` → `'30초를 생존하세요'`

### HUD.js
- `e3State`, `e3Kills`, `e3Elapsed` → `e2State`, `e2Kills`, `e2Elapsed`
- `[E3]` 텍스트 → `[E2]`
- 생존 표시: `Math.floor(e2Elapsed)/30` 으로 변경

### main.js
- `n.event === 'E3'` → `n.event === 'E2'`
- `eventSystem.e3State`, `e3Kills`, `e3Elapsed` → e2 variant
- `waveSystem.setEventMode('env_error', 3)` → spawn 제한 파라미터 제거

## T5: 보스 2페이즈 붉은 사각형 버그 수정

**파일**: `entities/Boss.js` — `render()` 의 phase2 overlay 블록

**문제**: BOSS_SPRITE 픽셀이 64×64 바운딩박스를 꽉 채우지 않아 빈 영역에 빨간 `fillRect`가 노출됨.

**수정 방법**:
```js
// 기존: ctx.fillRect(...) 단순 오버레이
// 변경: 스프라이트 위에 source-atop composite 적용

// 1. offscreen canvas에 스프라이트 먼저 그리기
// 2. globalCompositeOperation = 'source-atop' 으로 빨간색 얹기
// 3. 결과를 main ctx에 복사
```
- 구체적으로: `ctx.save()` → `ctx.beginPath()` → 스프라이트 픽셀마다 path 대신,
  PixelRenderer.drawSprite 호출 직후 `ctx.globalCompositeOperation = 'source-atop'` + `fillRect` 대신
  **OffscreenCanvas** 또는 별도 canvas element를 임시 버퍼로 사용해 스프라이트 영역에만 색 오버레이 적용
- 간단 대안: `PixelRenderer.drawSprite`에서 각 픽셀에 색조(tint) 파라미터 지원 추가

**권장 간단 구현**:
- `PixelRenderer.drawSpriteTinted(ctx, sprite, x, y, scale, tintColor, tintAlpha)` 추가
  - 각 픽셀 drawRect 전에 해당 픽셀 색을 tintColor로 블렌딩
- Boss phase2에서 `drawSprite` 대신 `drawSpriteTinted(ctx, BOSS_SPRITE, ..., '#cc0044', 0.3)` 사용
