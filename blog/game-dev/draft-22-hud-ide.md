# 22편: HUD를 IDE처럼 — Catppuccin 테마로 단색 UI 탈출

**📅 날짜:** 2026-04-25  
**🎯 분류:** fix (기존 기능 개선)  
**🔄 Cycle:** Phase 10.5 Cycle 1

---

## 문제: 단색 HUD, 게임처럼 느껴지지 않음

화면에 HP, 킬 카운트, 타이머를 표시하는 HUD는 있었지만, 너무 심심했다.

- **색상:** 흰 텍스트 1가지
- **배경:** 어두운 회색 1가지
- **텍스트 형식:** `"HP 20 / 100"`처럼 평면적

게임이라기보다는 터미널에 가까웠다. 심지어 텍스트도 한국어("킬:", "초") 섞여 있어서 통일성이 없었다.

---

## 풀이: VSCode 테마를 게임 UI에 그대로 담다

**아이디어:** 코딩할 때 보는 VSCode의 Catppuccin Mocha 테마를 HUD에 적용하면 어떨까?

IDE 에디터처럼 꾸미되, 게임 디버그 느낌도 살릴 수 있을 것 같았다.

```
변경 전:
HP 20 / 100
킬: 5 / 15
3:42

변경 후 (IDE 스타일):
hp = 20 / 100          ← 텍스트 색상 분리 (keyword, value, comment)
kills = 5  // target: 15   ← 마치 소스 코드처럼
elapsed = "3:42"       ← 변수명 + 값 형식
[E1] 활성 중  [E2] 클리어 ✓  ← 배지 박스 스타일로 상태 표시
```

### 색상 팔레트: Catppuccin Mocha

| 용도 | 색상 | 역할 |
|------|------|------|
| 키워드 (`hp`, `kills`, `elapsed`) | `#cba6f7` (자주색) | 변수명처럼 강조 |
| 값 (숫자, 문자열) | `#a6e3a1` (초록색) / `#f5c2e7` (분홍색) | 상태를 한눈에 |
| 주석 (`// target:`) | `#45475a` (회색) | IDE 주석처럼 희미하게 |
| 배경 패널 | `#1e1e2e` (진검은색) | VSCode 다크 모드 같은 분위기 |

### HP 바: 3단계 피드백

저체력 상황을 더 드라마틱하게 표현하고 싶었다.

- **50% 초과 (안전):** 녹색→청록색 그라데이션 + 부드러운 섀도우
- **25~50% (주의):** 주황색 그라데이션 + 경고 섀도우
- **25% 이하 (위험):** 빨강 + **펄스 애니메이션** (0.6~1.0 opacity, 800ms 반복)

낮은 체력일 땐 화면이 깜빡거리니 긴급 상황 느낌이 산다.

### 배지 스타일 이벤트 상태

기존엔 우상단에 텍스트만 나열했다. 이번엔 **박스**로 감쌌다.

```
[BOSS 등장] ← 펄스하는 빨간 테두리 박스
[E1 활성]  ← 주황색 배지
[E2 클리어]← 초록색 배지
```

투명도 + 색상으로 상태를 구분하니 UI가 훨씬 정성 있어 보인다.

---

## 구현: Canvas 2D API의 힘

게임은 DOM/CSS 없이 Canvas 2D API만 쓴다. 따라서:

1. **색상 상수화** → `static COLORS` 객체에 Catppuccin 팔레트 통합
2. **픽셀 단위 계산** → `ctx.measureText()`로 텍스트 폭 측정 후 세그먼트별 색상 적용
3. **애니메이션** → `Date.now()`로 현재 시간 기반 펄스 주기 계산

### 왜 이렇게 복잡한가?

Canvas에선 "이 부분만 색상 바꾸기"가 안 된다. 각 텍스트 조각을 개별 `fillText()` 호출로 그려야 한다.

```js
// "hp = 20 / 100" 을 세 부분으로 나누기
ctx.fillStyle = COLORS.keyword;
ctx.fillText('hp', x, y);
x += ctx.measureText('hp').width;

ctx.fillStyle = COLORS.text;
ctx.fillText(' = ', x, y);
x += ctx.measureText(' = ').width;

ctx.fillStyle = COLORS.text;
ctx.fillText('20', x, y);
// ... 이런 식으로 계속
```

번거롭지만, 이덕분에 IDE처럼 정확한 색상 제어가 가능하다.

## 변경사항 요약

| 항목 | 변경 전 | 변경 후 |
|------|--------|--------|
| **텍스트** | `HP 20 / 100` | `hp = 20 / 100` (신택스 컬러링) |
| **킬 표시** | `킬: 5 / 15` | `kills = 5  // target: 15` (IDE 형식) |
| **타이머** | `3:42` | `elapsed = "3:42"` (변수 형식) |
| **저체력 표현** | 색상만 변함 | 색상 + 펄스 애니메이션 |
| **이벤트 상태** | 텍스트만 | 박스 배지 (테두리 + 배경) |
| **전체 톤** | 단색 게임 UI | VSCode IDE 느낌 |

## 그래서 어떻게 됐나?

- ✅ HUD가 **눈에 띄게** 업그레이드됨
- ✅ 저체력 경고가 **더 극적**으로 느껴짐
- ✅ 게임보다 **개발 도구 느낌**이 났음 (의도대로!)
- ✅ 기존 로직 + 좌표는 100% 유지 (순수 시각 개선)
