## 추가 1: 게임 시작 화면

### 이전 상태

게임을 열면 바로 "무기를 선택하세요 (1 / 2 / 3)"가 떴다. 스토리도 없고, 조작법도 없고, 맥락도 없었다. 처음 보는 사람은 이게 뭔 게임인지 모른다.

### 설계

`state` 값에 `'intro'`를 추가했다. 흐름은 이렇게 된다:

```
'intro' → [Space/Enter] → 'menu' → [1/2/3] → 'playing'
```

인트로 화면에는 세 가지를 넣었다:
- **스토리**: "SSAFY 6반, 오후 11시. 동기들의 에러 메시지가 카톡으로 쏟아진다."
- **조작법**: WASD 이동, 자동 공격, Space로 알림 닫기, R키 재시작
- **목표**: 버그 처치 → 이벤트 클리어 → 보스 격파

한 가지 신경 쓴 건 `[Space] 시작` 텍스트의 깜빡임이다. `Date.now()`를 600ms 주기로 나눠서 on/off 처리했다. 별 것 아닌 것 같은데 이게 있고 없고 차이가 꽤 크다.

### handleMenuKey 방어 처리

`handleMenuKey`는 원래 state 체크가 없었다. 인트로 화면에서 1/2/3을 누르면 바로 게임이 시작될 수 있었다. `if (state !== 'menu') return;` 한 줄 추가로 해결했다.

## 추가 2: 이벤트 모달 캐릭터 대사

### 이전 상태

이벤트 발생 시 모달에는 "동기가 IndentationError 패닉! IndentationError 15마리 처치!" 같은 텍스트만 있었다. 누구의 이야기인지 알 수 없었다.

### 설계

`EVENT_MESSAGES`에 `character`와 `dialogueLines` 필드를 추가했다.

```js
E1: {
  triggered: '동기가 IndentationError 패닉! IndentationError 15마리 처치!',
  cleared:   '들여쓰기 지옥 탈출 성공!',
  character: '박진우',
  dialogueLines: [
    '아니 이거 또 이러네?',
    '지윤님 저 진짜 들여쓰기 오류 때문에 미치겠습니다..',
  ],
},
E3: {
  triggered: '동기의 IDE가 안 켜져! EnvError 처치 + 60초 생존!',
  cleared:   '파이참 위기 극복!',
  character: '이한정',
  dialogueLines: [
    '어? 이거 뭐야. 아.. 파이참 또 환경변수 꼬였네.',
    '지윤님 이것 좀 풀어주세요.',
  ],
},
```

`show()` 메서드에서 `triggered` 타입일 때만 `character`와 `dialogueLines`를 설정하고, `cleared`일 때는 null/빈 배열로 초기화한다.

렌더 쪽에서는 대사가 있을 때 모달 높이를 170 → 230으로 늘리고, 반투명 배경 위에 대사를 두 줄로 표시했다.

### TDD

테스트를 먼저 6개 추가했다:

```
show('triggered', 'E1') → character === '박진우'
show('triggered', 'E3') → character === '이한정'
show('cleared', 'E1')   → character === null
show('triggered', 'E1') → dialogueLines.length > 0
show('triggered', 'E3') → dialogueLines.length > 0
```

기존 7개 + 신규 6개 = 13개. 전체 누적 273개 GREEN.

## 돌아보며

이번 추가 사이클에서 코드보다 "어떤 정보를 언제 보여줄 것인가"를 더 많이 고민했다.

인트로 화면은 게임 설명이기도 하지만, 게임을 만든 맥락을 전달하는 장치이기도 하다. SSAFY, 6반, 밤 11시, 카톡 에러 알림 — 이 게임이 누구를 위해 만들어졌는지 처음 보는 사람도 금방 알 수 있다.

캐릭터 대사는 실존하는 동기들의 이름과 말투를 그대로 넣었다. 게임에서 그 이름을 보는 순간 "어, 나잖아" 하는 반응이 나올 것이다. 기술적으로는 별 것 없는 텍스트 렌더링이지만, 게임의 맥락을 살아있게 만드는 부분이다.
