## 그날 상황

Phase 5에서 10개짜리 태스크 목록을 실행하던 중 사용량 초과로 세션이 끊겼다. 몇 개는 완료됐는데 어디까지 됐는지 불분명한 상태. 다음 세션에서 이어서 하려니까 어디서부터 시작해야 할지 애매했다. 그러다 근본 문제를 깨달았다 — 사이클 완료 시에만 commit하는 규칙이 문제였다. 사이클 중간에 끊기면 완료된 태스크도 기록이 없으니 다음 세션이 처음부터 다시 하려 든다.

## 뭐가 나왔나

CLAUDE.md에 두 가지 규칙이 추가됐다.

첫 번째: 태스크 완료 시마다 `docs/phase-status.md` 체크 + `git commit && push`. 사이클 완료를 기다리지 않는다.

두 번째는 대화 중에 발견됐다. "phase5 진행해줘 입력 시 흐름이 어떻게 되냐"는 질문을 주고받다가 `docs/superpowers/plans/` 에 이미 `2026-04-16-phase5-plan.md` 가 존재한다는 걸 확인했다. 1~3단계(Superpowers→Gstack→GSD) 산출물이 이미 파일로 저장되고 있었던 것. 그래서 재진입 규칙도 추가했다 — 계획 파일이 있고 미완료 태스크가 남아 있으면 1~3단계를 스킵하고 다음 태스크 직행.

---

## 그래서 어떤 결정을 했나

CLAUDE.md에 두 섹션 추가:

1. **태스크 완료 시 필수 처리** — 태스크마다 phase-status.md 체크 + commit. 블로그 초안은 여전히 사이클 완료 시.

2. **사이클 진입 vs 재진입 판단** — `docs/superpowers/plans/` 파일 존재 여부로 판단. 파일 있으면 1~3단계 스킵, 계획 파일 로드 후 미완료 태스크 직행.

phase-status.md도 업데이트했다. T1·T2는 이미 완료됐지만 체크가 안 돼 있었는데 git 커밋 해시와 함께 반영했다.
