## 그날 상황

버그 몬스터들은 일정 주기마다 랜덤 방향에서 스폰되고, 플레이어를 향해 이동하다가 접촉하면 데미지를 준다. Phase 2의 핵심이었던 적 시스템이 막 완성된 시점이었다.

Phase 2 개발 중반이었다. 적 시스템까지 완성하고 다음 날 새 세션을 열었다.

AI가 물었다.

*"현재 어떤 작업 중인가요?"*

어제 한참 얘기했던 내용이었다. 어디까지 했는지, 다음에 뭘 해야 하는지. AI는 그걸 전혀 몰랐다.

---

## 입력한 프롬프트

프롬프트라고 할 것도 없었다. 그냥 세션을 새로 열었을 뿐이다.

AI는 대화 내역을 세션 간에 가져가지 않는다. 당연히 알고 있었다. 근데 막상 처음 겪으니 황당했다. 지금까지 쌓은 맥락을 처음부터 다시 설명해야 하는 상황.

---

## 뭐가 나왔나

그때까지 만들어둔 게 있었다. `docs/design.md`에 게임 기획이 있었고, `docs/phase-status.md`에 진행 상황이 적혀 있었다. 근데 AI한테 "세션 시작할 때 이걸 읽어라"고 말해둔 적이 없었다.

파일이 있어도 AI는 읽지 않는다. 읽으라고 명시해두지 않으면.

---

## 그래서 어떤 결정을 했나

`CLAUDE.md` 상단에 섹션을 하나 만들었다.

```
## 세션 시작 시 필독 (IMPORTANT)
- docs/phase-status.md — 현재 Phase 및 다음 작업 확인
- docs/design.md — 게임 전체 설계 확정본
```

이후로 새 세션을 열면 AI가 두 파일을 읽고 "Phase 3 Cycle 1 진행 중이군요"라고 바로 이어받는다.

파일 두 개가 AI의 단기 기억을 장기 기억으로 바꿔줬다. `phase-status.md`는 어디까지 했는지, `design.md`는 뭘 만들고 있는지. 세션이 바뀌어도 이 두 파일을 읽으면 맥락이 복원된다.

---

## 한 줄 회고

AI는 기억이 없다. 기억 대신 파일을 써야 한다. "당연히 알겠지"가 통하지 않는다.
