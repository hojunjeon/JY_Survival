## 입력한 프롬프트

> *"그럼 프롬프트일기를 스킬로 만들고 하나의 개발 사이클이 완료될 때마다 체크해서 블로그 글이 나올것 같으면 작성 제안하는 방식은 어때?"*

---

## 뭐가 나왔나

AI가 스킬 설계를 제안했다. 그리고 트리거 방식을 물어왔다.

*"사이클 완료 후 자동 실행할까요, 아니면 직접 호출할 때만 실행할까요?"*

그런데 내가 바로 잡았다.

> *"이 질문에는 모순이 있어. 단위 판단을 스킬 자체가 한다면서 스킬 트리거 방식은 왜 묻는거지?"*

스킬이 스스로 "쓸 만한 이벤트가 있었나"를 판단한다면, 트리거도 "사이클 완료 후 항상"으로 고정되는 게 맞다. 판단을 스킬에 위임했으면서 실행 여부를 다시 사람한테 묻는 건 반쪽짜리 자동화다.

---

## 그래서 어떤 결정을 했나

트리거 방식은 질문하지 않기로 했다. 수정/추가 사이클 완료 후 항상 실행, 스킬이 직접 판단해서 쓸 게 있으면 제안, 없으면 무시.

`~/.claude/skills/prompt-diary/SKILL.md`를 만들었다. 스킬이 하는 일:

1. `.claude/dev-log/`, git log, `docs/phase-status.md` 읽기
2. "주목할 만한 프롬프트 이벤트가 있었나?" 판단
3. 있으면 제안 블록 출력 + 승인 시 초안 작성
4. 없으면 아무것도 출력하지 않음

그리고 CLAUDE.md On-demand 섹션에 등록했다.
