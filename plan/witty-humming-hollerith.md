# 김지윤의 디버그 서바이벌 — 설계 & 구현 플랜

## Context

SSAFY 15기 서울 6반을 배경으로 한 뱀파이어 서바이벌류 브라우저 게임.
주인공 김지윤(전직 초등국어 교사, SSAFY 입과)이 몰려오는 버그들을 기술 스택 무기로 디버그하며 스테이지를 클리어한다.
이 프로젝트는 Superpowers + GSD + Gstack 워크플로우를 실증하는 Velog 개발일지 시리즈의 소재이기도 하다.

---

## 확정 설계

### 기술 스택
- **런타임**: 브라우저 (로컬 서버 불필요)
- **구현**: Vanilla JS + HTML5 Canvas, ES6 모듈
- **스프라이트**: Canvas로 직접 그리는 32×32 픽셀 아트

---

| 캐릭터 | 설명 |

### 핵심 게임 루프

```
무기 선택 → 버그 웨이브 생존
  → [시간 경과] 이벤트1 발생 (일시정지 + 동기 대사 알림)
    → 조건 달성 → 강화 재화 획득 → 재개
  → [시간 경과] 이벤트2 발생 (일시정지 + 동기 대사 알림)
    → 조건 달성 → 무기 획득 → 재개
  → [이벤트2 클리어 + 추가 시간 경과] 장선형 등장
    → 보스 처치 → 스테이지 클리어 → 무기 + 강화 재화 획득
퀘스트 Q1: 버그 100마리 처치 (이벤트 버그 포함) → 스탯 업그레이드 선택
```

### 스테이지 구조 (모든 스테이지 공통)

| 순서 | 내용 | 보상 |
|------|------|------|
| 퀘스트 Q1 | 버그 100마리 처치 (스테이지 전반 누적) | 스탯 업그레이드 선택 |
| 이벤트 1 | 시간 기반 트리거, 이벤트 버그만 등장 | 강화 재화 |
| 이벤트 2 | 시간 기반 트리거, 이벤트 버그만 등장 | 무기 획득 |
| 보스 장선형 | 이벤트2 클리어 + 추가 시간 후 등장 | 무기 + 강화 재화 |

**이벤트 발생 시**: 게임 일시정지 + 동기 대사 + 이벤트 알림 화면
**이벤트 클리어 시**: 게임 일시정지 + 보상 선택 화면 → 선택 후 재개

### 무기 시스템

| 무기 | 공격 방식 | 획득 방법 |
|------|---------|---------|
| Python | 360° 자동 투사체 (범위 넓음, 느림) | 시작 선택 |
| C/C++ | 전방 고속 단일 관통탄 | 시작 선택 |
| Java | 주변 오비탈 오브젝트 소환, 자동 공격 | 시작 선택 |
| Git | 범위 스턴/롤백 CC기 | 이벤트/보스 보상 |
| SQL | 광역 쿼리 공격 (쿨타임 김) | 이벤트/보스 보상 |
| JavaScript | 랜덤 방향 투사체 난사 | 이벤트/보스 보상 |
| Django | 전방 부채꼴 범위 공격 | 이벤트/보스 보상 |
| Linux/Bash | 화면 전체 궁극기 (쿨타임 매우 김) | 이벤트/보스 보상 |

**무기 강화**: 강화 재화 소모, 100% 성공률, 단계별 비용 증가
**무기 보유 한도**: 스테이지당 최대 4개
**스테이지 진입 시 무기 선택**:
- Stage 1: Python / C/C++ / Java 중 1개 선택 (시작)
- Stage N (N≥2): 소유 무기 중 N개 선택 (최대 4개)

### 적 (버그) 시스템

| 버그 | 이동 | 체력 | 접촉 데미지 | 특이사항 |
|------|------|------|------------|---------|
| SyntaxError | 직선, 보통 속도 | 보통 | 보통 | 기본형, 다수 등장, 빨간 느낌표 외형 |
| NullPointerException | 빠름 | 낮음 | 낮음 | 히트앤런, 반투명 유령 외형 |
| SegFault | 느림 | 높음 | 높음 | 탱커형, 깨진 조각 외형 |
| 회복 버그 | 빠름 (도망) | 낮음 | 없음 | 처치 시 HP 아이템 드롭 |
| EnvError (이벤트 전용) | 느림 | 높음 | 보통 | 미처치 시 이동속도 감소 장판 생성 |

> 디버프 시스템 (피격 시 이동속도/공격속도 감소 등)은 MVP 이후 추가 예정.

### 보스 — 장선형

| 페이즈 | 조건 | 행동 |

- 도발 (랜덤): *"지윤님 맨날 저 놀리더니 이제 후회하시겠죠?! 이번엔 제 차례예요!!"*
- 처치: *"말도 안 돼... 이건 진짜 억울하다고요!! 지윤님 반칙이에요!!"*
- 추가 랜덤: *"왜 저만 이래요!! 분명히 저도 열심히 했는데!!"*

## 구현 플랜

### 파일 구조

```
/
├── index.html              # 진입점, Canvas 마운트
├── main.js                 # 게임 부트스트랩
├── core/
│   ├── Game.js             # 메인 게임 루프 (requestAnimationFrame)
│   ├── Canvas.js           # 렌더링 유틸 (clearRect, drawImage 등)
│   └── Input.js            # 키보드 입력 (WASD / 화살표)
├── entities/
│   ├── Player.js           # 김지윤: 이동, HP, 무기 보유 목록
│   ├── Enemy.js            # 버그 기반 클래스 (이동, HP, 데미지)
│   ├── Boss.js             # 장선형: 2페이즈, 대사, 투사체 패턴
│   └── Projectile.js       # 투사체 (위치, 속도, 데미지, 충돌)
├── weapons/
│   ├── WeaponBase.js       # 무기 기반 클래스 (공격 주기, 강화 레벨)
│   ├── Python.js
│   ├── C.js
│   ├── Java.js
│   ├── Git.js
│   ├── SQL.js
│   ├── JavaScript.js
│   ├── Django.js
│   └── LinuxBash.js
├── stages/
│   ├── StageBase.js        # 스테이지 기반 (이벤트 큐, 보스 트리거)
│   └── Stage1.js           # 파이썬 기초 스테이지 정의
├── systems/
│   ├── EventSystem.js      # 이벤트/퀘스트 상태, 트리거, 완료 처리
│   ├── WaveSystem.js       # 버그 웨이브 스폰 (시간 기반, 이벤트 전환)
│   └── UpgradeSystem.js    # 강화 재화 관리, 무기 강화, 스탯 업그레이드
├── ui/
│   ├── HUD.js              # HP바, 재화, 무기 슬롯, 퀘스트 진행도
│   ├── EventModal.js       # 이벤트 발생/클리어 모달 (일시정지)
│   └── Menu.js             # 시작 화면, 무기 선택, 스테이지 클리어 화면
└── sprites/
    └── PixelRenderer.js    # Canvas API로 32×32 픽셀 스프라이트 직접 그리기
```

### 구현 순서 (원자 태스크)

#### Cycle 1 — 코어 & 플레이어
1. `index.html` + Canvas 마운트
2. `Game.js` 게임 루프 (update / render 분리)
3. `Input.js` WASD 입력
4. `Player.js` 이동, HP, 충돌박스
5. `Canvas.js` 기본 렌더링 유틸
6. `PixelRenderer.js` 김지윤 32×32 스프라이트

#### Cycle 2 — 적 & 웨이브
7. `Enemy.js` 기반 클래스 (SyntaxError, NullPointerException, SegFault)
8. `WaveSystem.js` 시간 기반 스폰
9. 충돌 감지 (플레이어 ↔ 적)
10. 회복 버그 + HP 아이템 드롭

#### Cycle 3 — 무기 시스템
11. `WeaponBase.js` + `Projectile.js`
12. Python, C/C++, Java 구현 (시작 무기 3종)
13. 무기 선택 화면 (`Menu.js`)
14. 충돌 감지 (투사체 ↔ 적)

#### Cycle 4 — 이벤트 & 퀘스트
15. `EventSystem.js` (트리거, 일시정지, 모달)
16. E1 이벤트 (IndentationError 웨이브)
17. E3 이벤트 (EnvError 웨이브 + 60초 생존)
18. Q1 퀘스트 (버그 100마리 카운팅 + 스탯 업그레이드)

#### Cycle 5 — 보스 & 클리어
19. `Boss.js` 장선형 (2페이즈, 투사체 패턴, 대사)
20. 보스 등장 트리거 로직
21. 스테이지 클리어 판정 + 보상 화면

#### Cycle 6 — 강화 & HUD
22. `UpgradeSystem.js` (강화 재화, 무기 강화)
23. `HUD.js` (HP, 재화, 무기 슬롯, Q1 진행도)
24. `Stage1.js` 전체 통합 테스트

#### Cycle 7 — 문서화
25. `/blog/draft-1-기획.md` 브레인스토밍 편 작성
26. `/blog/draft-2-코어개발.md` 코어 구현 편 작성

### 검증 방법

- `index.html`을 브라우저에서 직접 열어 실행 (로컬 서버 불필요 — ES6 모듈은 `<script type="module">` 사용, 단 file:// 프로토콜에서 CORS 발생 시 `npx serve .` 사용)
- 플레이어 이동, 적 스폰, 충돌, 이벤트 트리거, 보스 등장 순서로 골든 패스 확인
- 사망 → 스테이지 재시작 플로우 확인
- 무기 최대 4개 제한 확인
