# 수정 및 보완 사항

## #수정

- [ ] **무기 기믹 재구현 — Python**: 현재 느린 단순 투사체. 초록색 뱀 궤적이 적→적 사이를 전이(Chain Lightning)하는 방식으로 교체. 전이 횟수 최대 3회, 전이 시 짧은 경직(freeze) 부여.
- [ ] **무기 기믹 재구현 — C/C++**: 현재 단순 직선 관통탄. 발사 전 0.5초 조준선(점선) 표시 후 화면 끝까지 관통하는 레이저(레일건)로 교체. 발사 시 수직 Screen Shake 적용.
- [ ] **무기 기믹 재구현 — Java**: 현재 단순 오비탈 구체. 지정 위치에 블랙홀 소환 → 주변 적을 중심으로 흡입(위치 강제 이동) → 일정 시간 후 폭발 광역 대미지. 흡입 중 적 이동속도 감소.
- [ ] **무기 기믹 재구현 — Git**: 현재 이펙트 없는 빈 껍데기. 발동 시 현재 위치에 분신(Branch) 생성 → 3초 후 분신↔본체 사이 직선 상 전체 적에게 대미지(Merge). "Merge Conflict Resolved!" 플로팅 텍스트 연출.
- [ ] **무기 기믹 재구현 — SQL**: 현재 단순 광역 공격. 화면 중앙에 표적 표시 → 1초 후 상단에서 `DROP TABLE` 블록 낙하 → 착지 시 전체 화면 플래시 + 넉백. 쿨타임 유지(15초 이상).
- [ ] **무기 기믹 재구현 — JavaScript**: 현재 무작위 난사. 나선형으로 퍼지는 소용돌이 에너지장 생성 → 범위 내 적의 이동 방향을 강제 전환(밀어내기). 괄호 `( { } )` 파티클 연출.

## #추가

- [ ] **신규 몬스터 — Race Condition (샴쌍둥이 버그)**: 동일 외형의 몬스터 2마리가 실선으로 연결되어 등장. 한 마리만 처치 시 1초 후 풀피 부활. 두 마리를 1초 이내 연속 처치해야 제거 완료. 광역 무기 활용 유도.
- [ ] **신규 몬스터 — Memory Leak (메모리 누수 슬라임)**: 시간이 지날수록 크기(hitbox, 스프라이트) 증가. 이동 경로에 '가비지' 오브젝트 잔류 → 플레이어 접촉 시 대미지 + 이동속도 감소. 일정 크기 이상이면 스폰 차단. 빠른 처치 유도.
- [ ] **신규 몬스터 — Infinite Loop (가두리 양식러)**: 플레이어 주변을 원형으로 공전하며 투사체 발사. 투사체는 플레이어 주변에 '코드 벽' 오브젝트 생성 → 일정 개수 이상 쌓이면 플레이어 이동 제한. 본체 처치 시 벽 전체 제거.
- [ ] **신규 몬스터 — Input Mismatch (컨트롤 반전술사)**: 보라색 경고파 투사체 발사. 피격 시 3초간 WASD 방향키 반전(W↔S, A↔D). 공격 전 전조 현상(보라색 충전 이펙트 0.7초) 표시 필수 — "피했어야 했는데"의 납득 필요.
- [ ] **신규 몬스터 — Library Dependency (패키지 버퍼)**: 택배박스 형태 몬스터. 반경 내 다른 몬스터 전체에 방어력 5배 버프 부여(또는 무적). 패키지 몬스터 처치 시 버프 즉시 해제. 플레이어가 점사 우선순위를 판단하도록 유도.

## #제안

- [ ] **웨이브 구성에 신규 몬스터 통합**: Race Condition은 중반(웨이브 5+), Memory Leak은 초반부터, Infinite Loop/Input Mismatch는 후반(웨이브 8+), Library Dependency는 보스 직전 구간에 등장하도록 WaveSystem.js 웨이브 테이블 업데이트.
- [ ] **무기 기믹 구현 후 밸런스 검토**: 각 무기 기믹 구현 완료 시 다음 Gemini 세션에서 "12년 차 게임 개발자" 페르소나로 밸런스 평가 재요청.
