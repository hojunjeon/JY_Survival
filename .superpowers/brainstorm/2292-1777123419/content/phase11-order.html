Phase 11 — Cycle 순서 방식

5개 Cycle을 어떤 순서로 진행할지 선택해줘

A

UI 우선 → 게임플레이 확장

Cycle 1 A1 미구현 화면 7개 → Cycle 2 A3 무기 레벨업 이펙트 → Cycle 3 B3 보상 시스템 → Cycle 4 B1 무기 시너지 → Cycle 5 B2 스테이지 추가

장점

  • 화면 먼저 완성 → 이후 기능 개발 시 UI 있음
  • 블로그 소재: 레퍼런스 재현기 스토리라인

단점

  • A1이 가장 큰 작업 → 시작이 무거움

★ 추천 — 레퍼런스 따라잡기를 목표로 했으므로 UI 먼저가 자연스러움

B

게임플레이 먼저 → UI 나중에

Cycle 1 A3 이펙트 → Cycle 2 B3 보상 시스템 → Cycle 3 B1 무기 시너지 → Cycle 4 A1 UI 화면 → Cycle 5 B2 스테이지 추가

장점

  • 작은 것부터 → 모멘텀 쌓기 쉬움
  • 게임플레이 검증 후 UI 작업

단점

  • UI 없는 상태로 기능 개발 → QA 어색함
C

의존도 순서 (논리적)

Cycle 1 B3 보상 시스템 → Cycle 2 B1 무기 시너지 → Cycle 3 A3 이펙트 → Cycle 4 A1 UI 화면 → Cycle 5 B2 스테이지

장점

  • 보상→시너지→이펙트→UI 순: 각 단계가 이전 것 위에 쌓임

단점

  • UI가 제일 늦어 레퍼런스 비교가 마지막에나 가능
